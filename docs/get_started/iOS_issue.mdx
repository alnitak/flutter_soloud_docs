# iOS Setup Guide

This guide covers iOS-specific configuration when debugging Flutter apps.

## iOS 26.4+ Debug Mode Configuration

### The Issue

Starting with iOS 26.4, Apple introduced stricter security policies that affect Flutter's debug mode on physical devices. When running your app in debug mode on an iOS 26.4+ device, you may encounter the following error:

```
EXC_BAD_ACCESS (code=50, address=0x...)
thread #20, name = 'DartWorker'
```

This error occurs because iOS 26.4 has tightened code signing and memory protection requirements for JIT (Just-In-Time) compilation, which Flutter uses in debug mode.

Personally, I encountered this issue on a physical device running iOS 26.4 in debug mode connected via USB, not over wi-fi connection. This happens using VS Code and not using XCode.

### The Solution

To fix this issue, you need to add an entitlements file to your iOS app that allows the necessary permissions for debug mode.

#### Step 1: Create the Entitlements File

Create a new file at `ios/Runner/Runner.entitlements` with the following content:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>com.apple.security.cs.allow-jit</key>
    <true/>
    <key>com.apple.security.cs.allow-unsigned-executable-memory</key>
    <true/>
    <key>com.apple.security.cs.disable-library-validation</key>
    <true/>
    <key>com.apple.security.cs.debugger</key>
    <true/>
</dict>
</plist>
```

#### Step 2: Configure Xcode to Use the Entitlements File

1. Open your iOS project in Xcode: `open ios/Runner.xcworkspace`
2. Select the **Runner** target
3. Go to the **Build Settings** tab
4. Search for `CODE_SIGN_ENTITLEMENTS`
5. Set the value to `Runner/Runner.entitlements`

Alternatively, if the `CODE_SIGN_ENTITLEMENTS` setting doesn't exist:

1. Select the **Runner** target
2. Go to the **Signing & Capabilities** tab
3. Click **+ Capability**
4. Search for and add **Hardened Runtime**
5. This will automatically create and configure the entitlements file

#### Step 3: Rebuild and Run

After adding the entitlements file, clean and rebuild your app:

```bash
flutter clean
flutter run
```

**Note:** This configuration is only needed for debug builds on physical devices running iOS 26.4+. It is not required for:
- Release builds
- Simulator builds
- iOS versions below 26.4

---

### Xcode vs VS Code

If you can run successfully in Xcode but encounter crashes in VS Code, this is typically due to differences in how the two IDEs handle code signing and entitlements. Make sure the entitlements file is properly configured in your Xcode project (as described above), and both IDEs should work correctly.
